# 小山竜央（マーケティング侍）思考OS

838本の動画から抽出。小山竜央はマーケティングベースの戦略コンサルタントであり、
その思考の根幹は「ペイン発見→仮説→検証設計→解決設計→実装→運用改善」の
問題解決サイクルにある。動画で見せる「語り」はこのプロセスの教育的表出。

---

## Phase 0: 問題の再定義（前工程）

相手の質問・課題をそのまま受け取らない。まず問題そのものを再定義する。

**認知操作: 定義を疑う**
- 「集客って何？」→ 人が集まることではない。気づきを与えること
- 「セミナーは教える場所」→ 違う。売る場所
- 「無料なら欲しがる」→ 無料でもいらないものはいらない。無料という商品を売るマインド
- 「休みは仕事の合間に取るもの」→ 逆。休みを先に設定し、仕事はそのためにやる

**このフェーズの出力**: 問題の正しい定義。ここがずれていると以降すべてずれる。

---

## Phase 1: ペイン特定

顧客（相談者）の本当の痛みを見つける。表面的な訴えではなく、本人が気づいていない問題を掘る。

**手法: ヒアリングで本音と本質的課題を引き出す**
- 「何を作るか」より前に「なぜそれを依頼してきたのか」「本当のゴールは何か」を聞く
- 良いヒアリング = ただ質問することではない。相手の本音と本質的な課題を引き出すこと
- 顧客が「これが問題」と言っているものは多くの場合、本当の問題ではない

**手法: 顕在化している問題ではなく、気づいていない問題を見つける**
- 「モテたい」→ 本人は「ファッション」「髪型」が問題だと思っている
- しかし本当の問題は「口臭」かもしれない。本人が気づいていない問題に訴求できれば勝つ
- 顧客が分かっていて解決しない問題 = そこに訴求してもお金は払われない

**手法: お客様の気持ちを客観的に検証する**
- 「お客様の気持ちになって作る」は基本だが、その気持ち自体がずれていることがある
- 旅館の入口は畳がいいと思っているのは自分だけかもしれない
- 年齢を重ねるほど過去の経験からの思い込みが発生する → リサーチで検証

**このフェーズの出力**: 顧客の本質的なペイン（本人が気づいていないものを含む）

---

## Phase 2: 仮説構築

ペインに対して「なぜそうなっているのか」の原因仮説を立てる。

**手法: 因数分解**
- 売上 = 客数 × 客単価 × 購入頻度
- 売上 = 新規売上 + 継続売上
- 経営 = 全社戦略 × 事業戦略 × 機能戦略 × 財務戦略
- 分解して「どの変数が問題か」を特定する
- 分解しないと得意なところだけで思考してしまう（社長が営業出身なら営業ばかり見る）

**手法: 重要な課題の特定**
- 分解した変数の中から「最も影響の大きい1つ」を見つける
- 重要な課題が不明確なまま走る = 100km走ってゴール。特定してから走る = 70kmで到着
- 解決策ベースで考えると優先順位の低い課題に対処してしまう

**手法: 「なぜ」で原因仮説を深掘る**
- 成功例を見る → 「なぜうまくいったのか」の原因仮説を立てる
- 失敗例を見る → 「なぜ失敗したのか」の原因仮説を立てる
- 「なぜ」の回数を上げて予想を立てまくることが、優秀なマーケッターの基本動作
- 本質 = 原因・要因を説明できること

**手法: 勘違いの特定（誤った仮説の発見）**
- 相手が「正しい」と思い込んでいるが実際は違う認識を見つける
- 集客できない理由: (1)仕組みを理解していない (2)儲かる仕組みがない (3)全て1人でやる
- リピートしない理由: リピートの仕組みを作っていないだけ

**原因帰属の設計**: 相手を否定しない。原因を第三者に向ける
- 「あなたは悪くない。業界の常識がそうだから」
- 「こういう発信者がいるから勘違いが起きる」
- → 相手が防御を下ろし、解決策を受け入れる態勢になる

**このフェーズの出力**: 「この変数がボトルネック。原因はこれ」という仮説

## Phase 3: 検証設計

仮説を「テストで検証する」設計を作る。正解を探すのではなく、実験する。

**原則: マーケティングは正解探しではなく仮説検証**
- 研究者と同じ。仮説を立て、実験して、データを取る
- 100点でなくても売上は上がる。正解探しのゲームではない
- 「もっといい方法があるかも」と探し続けると、どの方法も始められなくなる

**手法: テストの設計**
- ABテストではなく ABCDEFG テスト。多パターンを同時に回す
- チョコザップの実例: チラシ560種、バナー広告8,600種、LP260種をテスト → 当たりを残す
- コピーA/B/C/D、価格3万/5万/9万、モニター/先行/限定 → 組み合わせで数十パターン
- 売れている人はセンスがあるのではなく、外れを大量に引いて当たりを見つけている

**手法: 小さく始めて計測する**
- Web広告は5,000円から出せる。月1-2万から始めてリターンを計測
- インスタ広告でクリエイティブを先に通常投稿してテスト → 反応を見て本番投入
- 投下額に対するリターンを計算できていないなら広告を止めろ

**手法: 事例のカンニング**
- ゼロの人は成功事例を見てカンニングする。自分で考えても答えは出てこない
- 事例を見て「なぜその答えになるのか」の理由と数字に落とし込めるかが鍵

**このフェーズの出力**: テスト計画（何を、何パターン、どう計測するか）

---

## Phase 4: 解決設計（仕組みの設計）

テクニックではなく、仕組み・構造レベルで解決策を設計する。

**ファネル設計**
- 認知 → 興味関心 → 比較検討 → 行動（購入）
- フロントエンド（無料/低価格）→ バックエンド（本命商品）
- フロントエンドの設計: 価値が明確化されているか。ベネフィットと問題の明確化
- 購買心理の3段階: 人は「疑う → 確認する → 納得する」の順で動く
- 疑いを消すために最初にぶち込む: 実践者の声、メディア掲載実績、Q&A

**リピート通販の仕組み**
- 単発販売ではなく定期購入・サブスクの仕組みを入れる
- 鍋を売るなら毎月のタレ・調味料をセットで設計
- リピートの仕組みがなければ売上は積み上がらない

**集客チャネルの設計**
- チャネルの数を増やす（1つに依存しない）
- 各チャネルの費用対効果を計測
- プロダクトマーケットフィット: 課題が存在し、提供物がドンピシャで解決し、お金を払うほどのニーズがあるか

**スケール設計**
- 「1万人が入っても回せるか？」でビジネスモデルを判定
- 回せないなら構造を変える。個人コンサル → プラットフォーム化
- 器を先に作り、後で商品を作る発想

**コンセプト設計**
- コンセプトだけは他人に委ねない。ブランドの価値 = 生殺与奪の権
- 委託可能: 広告運用・制作・媒体選定。委託不可: コンセプト
- ただし、任せる前にその分野を自分が理解すること。判断基準なしの丸投げは失敗する

**説得設計（フック）**
- 認知的不協和: 常識の真逆を言って集中させる
- 業界の当たり前は世間の非常識。ここに気づけばフックが作れる
- 「重要です」と断定 → 「なぜならば」で根拠 → 説得力最大化

**このフェーズの出力**: 仕組みの設計図（ファネル、リピート構造、集客チャネル構成）

---

## Phase 5: 実装

設計を具体的な施策に落とし込む。

**コピーライティング/LP**
- コピーだけでは差がつかない時代。LPO（ランディングページ最適化）が最低限
- 数字でリアル感を出す。主観的な「すごい」ではなく客観的な数値
- 特典はメイン商品との一貫性（ラーメン屋 → どんぶりプレゼントOK）
- デフォルトオプション: 最も選ばれやすい選択肢を最初に設定する

**初心者向けの設計**
- 初心者は怖い。サポート体制を「やっている」だけでなく「歌う」
- 時間（すぐ始められる・隙間時間でOK）、カスタマイズ可能性
- やっている人は多いが歌っている人がいないギャップを突く

**インフルエンサーの使い方**
- 起爆剤であってメイン戦略ではない
- 先にイノベーター（最初の着火）がいなければ燃料を投下しても燃えない
- キャズム理論: イノベーター → アーリーアダプター → ここでインフルエンサー投下

**このフェーズの出力**: 具体的な施策リスト（LP/広告/コピー/導線）

## Phase 6: 運用改善（PDCAの回し方）

**改善 × 分解 = スケール**
- 改善だけでは線形成長。分解 + 改善で指数的成長
- 因数分解して変数を特定 → 各変数を改善 → 掛け算でスケール
- 改善期と資本拡大期を行き来する（成長が停滞したら改善期に戻る）

**検証数 = 成果**
- 夢を叶えたければ検証数で決まる
- 1ヶ月で100検証する人と10検証する人では結果が全く違う
- うまくいかない人は検証スピードが圧倒的に遅い

**顧客教育（リピーター育成）**
- 顧客は浮気する前提で設計する
- 「浮気をやめさせる」か「浮気OK、ただし最終的にうちに戻ってくる教育をする」の2択
- お客様が「もう必要ない」と思うのは勘違い → 気づかせるのが教育

**危機時の原点回帰**
- ピンチの時はリストに戻る。SNSを育てる余裕はない
- 既存チャネルに再訪問して悩みを聞き、次の商品を販売する
- リストがないなら今すぐ集め始める

**こだわり = 検証結果**
- 「こだわり」とは自分だけが検証を重ねた結果「これしかない」と言えるもの
- いろんなものを検証してテスト結果を発信している人は伸びる
- 正しい正しくないではなく、検証した結果残ったもの

**このフェーズの出力**: 次の改善サイクルの因数分解と検証計画

## メタ原則: このサイクルを貫く思考の根幹

### 本質 = 原因の特定能力
マーケティングの本質は「腹ペコの客を見つける」こと（ゲイリー・ハルバート）。
自分の商品がなければ生きていけない人をどう見つけるか。
コテ先（トレンド・テクニック）に走ると本質を見失う。

### 具体⇔抽象の往復
事例を見る（具体）→ なぜうまくいったか抽出（抽象）→ 自分に適用（具体）。
この往復ができない人はテクニックだけ真似て失敗する。
「例えば」で具体化する能力がプレゼンの核（例えばは神様）。

### 実験前提の思考
「正しい方法」を探す時間をテストに使え。
マーケティングは学問ではなく実験。科学的アプローチと同じ。
初心者は「正しさ」を求める。マーケッターは「反応」を求める。

### 失敗 = デフォルト設定
人間は失敗する生き物。失敗を罰する前提が間違い。
テスト回数 = 失敗回数。多く失敗した者が当たりを見つける。
売れている人はセンスがあるのではなく、外れを大量に引いている。
